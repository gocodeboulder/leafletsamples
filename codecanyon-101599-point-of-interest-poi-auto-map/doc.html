1) Introduction - What is POI Auto Map - top

Poi Auto Map combines the power of Google's Maps and Places APIs to give you a simple way to add highly interactive Point of Interest maps to your website. All you need to do is define a starting address and a list of point of interest categories and POI Auto Map does the rest. Move the map and watch the points update. You can also define a database connection from which to source your POI points.

2) System Requirements - top

POI Auto Map is a Javascript script that runs mainly within the browser, as well as some basic PHP for database and Places processing. It has only been tested in LAMP environments, however it may be possible to use in other environments as long as PHP / cURL is available.

You need to add a <script> tag to link to Google Maps API V3: 

<script type="text/javascript" src="http://maps.google.com/maps/api/js?sensor=false"></script>

You need to add a <script> tag to link to Jquery: 

<script src="http://code.jquery.com/jquery.min.js" type="text/javascript"></script>

The included index.html should be referred to to see all required script tags.

2.1) Google Places API - top

In order to utilize Google Places Data, you will need to sign up for a Google Places API Key by following the instructions here: http://code.google.com/apis/maps/documentation/places/#Authentication

Your API Key needs to be added to the /db/jsonproxy.php file:

$api_key = "your-key-goes-here";

Marker Icons

You will need to provide an indivudual png icon for each category. You can make your own, purchase one from envato such as this - http://graphicriver.net/item/map-marker-mega-pack-66-icons/86505 , or download a free icon library such as the one used in the demo from here: http://mapicons.nicolasmollet.com

3) Installation - top

Copy the contents of the zip file to a directory off the root of your site. You should have the following directories:

  • css - contains the default styling for the demo
    • images - contains images used in the demo
  • icons - Empty (Put your marker Icons here)
  • js - Contains the main poi.js file
  • demo - contains demo files used in the index.html demo.
  • images - content images used in the demo
  • print - files used for printable Driving Directions
  • db - files used for database connector, bulk geocoding and json proxying (Google Places)

 

4) Usage - top

POI Auto Map is driven by some simple HTML and Javascript the index.html file is a good example of the code required.

4.1 Javascript Configuration - top

The following configuration items are required to be made in the poi.js file to configure your POI Auto Map install.

  // Map Configuration
	var defaultAddress = "111 Main St, Smithtown, State"; - Starting Address for initial display and Main Marker
	var defaultHTML = "<strong>Baloon Content</strong>";  - HTML Content to be displayed in Main Marker popup 
	var infoHTML = "<h4>Help to use this</h4>"; - HTML Content to be displayed in Main Marker popup 
	var defaultZoomLevel = 16; - Initial Zoom Level of map, Larger Number = Higher Zoom
	var searchControls = true;- Display the search for an address nar true / false
	var mapDivID = "map"; - the defult div ID to use for the map
	var categoriesList = "poiList"; - the ID of the Unirdered list (UL) to source categories from
  var directionsMode = 'DRIVING'; - change to 'walking' WALKING for walking directions
  var infoWindowDivID = ""; - The ID of a div that can be used instead of the standard infoWindow bubble

  // Display Photos In Info Window - Make Sure You Add Your Google Places API Key to db/imageproxy.php
  var showPhotos = true; 
  var maxPhotoWidth = 150; 
  // Display Google Places Rating
  var showRating = true; 

  // Geolocation features
  var useGeolocation = false;  - Enable geolocation for browsers that support it. 
  var autoGeolocation = false; - Send geolocation able browsers to their current location by default

  // Database Bounds Search Extension
  var extendLat = 0.05; - Boundary extension for database searches. Distance outside visible map.
  var extendLng = 0.005;
	var dbPath = "db/"; - Path to the db.php and jsonproxy.php

  // Places Search Radius (metres)
  var searchRadius = 2000; - distance in metres from main marker to search places api

  // Main Marker Configuration 
	var mainIconWidth = 43; - Width of Main Point Icon Marker
	var mainIconHeight = 50; - Height of Main Point Icon Marker
	var mainAnchorPointX = 17; - Horizontal Point of the Main Marker Icon to use as attachment point
	var mainAnchorPointY = 49; - Vertical Point of the Main Marker Icon to use as attachment point

	// Icon Configuration
	var iconPath = "/icons/"; - Relative Path to the Icons Directory
	var iconWidth = 32; - Width of POI Icon Markers
	var iconHeight = 37; - Height of POI Icon Markers
	var anchorPointX = 16; - Horizontal Point of the Marker Icon to use as attachment point
	var anchorPointY = 34; - Vertical Point of the Marker Icon to use as attachment point
	var iconScaleWidth = 24; - width to display icons
	var iconScaleHeight = 24; - height to display icons
	var useGoogleIcon = false; - Use Googles icons built in to the Places API

4.2 < Head Tags> - top

The following items must be placed in the <head> area of your page:

<script type="text/javascript" src="http://maps.google.com/maps/api/js?sensor=false"></script>
<script src="http://code.jquery.com/jquery.min.js" type="text/javascript"></script>
<script src="js/poi.js" type="text/javascript"></script>

You can also link to the default CSS Style sheet in the <head>:

<link rel="stylesheet" type="text/css" href="css/style.css"/>

4.3 Page Level Configuration - top

Some uses of POI Auto Maps may require that each page has different location, balloon HTML and zoomlevel. This can be acheved by adding the following before the po.js script tags:

<script type="text/javascript">
var poiAddress = "Glendale Rd, Briar Hill, Vic";
var poiHTML = 'Briar Hill, Vic';
var poiZoomLevel = 15;
</script>

4.4 HTML Setup - top

For your Auto POI Map to work, you need to add some HTML to the <body> of your page. At a minimum, you must have the following:

<div id="map"></div>
<ul id="poiList">
	  <li class="hidden" data-title="establishment" id="shoe_store" name="shoes">Shoe Stores</li>
</ul>

The map div is the container where the map will be rendered.

The poiList ul and its child li's define the list of POI categories - click for more info. You can add as many child li's as you wish to have categories.

You should refer to the index.html for a full, styled example.

5) POI Category Setup - top

The POI categories displayed on your maps are determined by the li items in the poiList UL (click for more info) :

<div id="map"></div>
<ul id="poiList">
	  <li class="hidden" data-title="establishment" id="shoe_store" name="shoes">Shoe Stores</li>
	  <li data-title="police" id="police">Police</li>
</ul>

Each li needs the following parameters to create the markers:

  • title (required) - This is the supported Places API type that will be searched for to find this categories points of interest. This must be one of the supported types in this list http://code.google.com/apis/maps/documentation/places/supported_types.html
  • id (required) - The unique identifier for this category. Must be unique. This also defines the marker icon to be used (click for more info)
  • class (optional) - This defines whether this categories POI markers are shown when the map is loaded. Leave blank or omit to have the markers shown on load.
  • name (optional) - A term to be matched against the names of Places. Results will be restricted to those containing the passed name value. When a name is included, the area being searched may be broadened, to ensure a suitable number of results.

5.1) Category Search Box - top

You can also add an item to your unordered list that defines where the user defined category search box will be placed.

 <li data-title="" id="user">Search</li>

This should be added as above with no alteration. You will need to provide a user.png file in the icons directory for use as the Pin for this category.

6) Marker Icons Configuration - top

Poi Markers

Each POI category requires its own png format icon to use as a marker on the map. Each png file must be named the same as the ID tag for each category. eg

 <li class="hidden" data-title="police" id="police" >Police Stations</li>

You must have a corresponding police.png file in the icons directory.

You can also add a category-shadow.png file in the icons directory. eg for the example above, you would provide a police-shadow.png file in the icons directory.

You need to define the size and anchor of the icons used in the javascript config. All marker icons must be the same size.

Main Marker Icon

You can provide a png file to use as the main marker that is used to mark the initial address. This file must be named pin.png. You can also provide a pin-shadow.png file to use as the markers shadow. If you do not provide a pin.png, no marker will be added to the map.

You need to define the size and anchor of the main icon used in the javascript config. This can be a different size to the POI category marker icons..

8) Database Connector - top

POI Auto Maps now includes a database connector so that you can add POI markers from your own database, as well as Google Search results.The database used can be anexisting mySWL database that you are using or it can be one that you create solely for this purpose. There is no tool included in this package for adding / editing database data. You can create a demo database by using the db/poi.sql file to create it.

You must be running PHP 5.1 or better and be using mySQL. You also must have PHP PDO installed on your server.

8.1 - Database Connector Setup

In order to use the database connector, you will need to open and edit the db/db.config.php file with your information (edit the areas in bold, inside the quotaion marks):

'hostname'	=>	'localhost',  - The server that your database is located on
'username'	=>	'db_user',  - The user name to be used when connecting to the database
'password'	=>	'password',  - The password to be used to connect
'dbname'	=>	'poi_db',  - The name of the database to be used
'tablename'	=>	'poi'  -  The table that holds the POI information

You will also need to define the fields in the database to be used by editing db/db.config.php file (more info on the field and their structure below):

'latitude'	=> 'lat', - The field in your table that has the points latitude
'longitude'	=> 'lng', - The field in your table that has the points latitude
'title'		=> 'title', - The field in your table that has the main title of the popup window
'address'	=>	'address', - The field that holds the POI's address
'url'		=> 'url', - The field that has the URL that you want the title to link to 
'html'		=> 'html', - The field that contains the HTML to be used in the popup
'category'	=> 'category', - The field that contains the category for the point. Also defines the default marker.
'icon'		=> 'marker' - The field that contains the costom marker value.

8.2 - Database Structure

You can use an existing database / tanble or can create a new one just for POI's. Either way, the table that you use must contain the following fields:

Field Type Length Decimal Information
latitude float 15 10 The latitude of your Point of Interest - required for the point to be shown
longitude float 15 10 The longitude of your Point of Interest - required for the point to be shown
title text     The text shown as the title in the marker's popup - may be left blank.
address text     The Location's address - may be left blank
url text     The url that the title links to - may be left blank or '#'
html text     The html to be shown in the popup window - may be left blank
category text     The category that this point belongs to. Also used as the default icon for the group - required for the point to be shown
icon text     The filename (minus the png extensions) in the icons directory to be used for this marker only - leave blank to use the category marker

8.3 Category HTML

To define a category to be sourced from your database, you need to add a category to the poiList. eg

      <li data-title="db:property" id="property">Property For Sale</li>

This tells POI Auto Map to search the database for all markers in the currently viewed map bounds for records with property as the category

You must have a corresponding category.png file in the icons directory or be using a custom marker as defined in the icon .eg for the example above, you would provide a property.png file in the icons directory.

You can also add a category-shadow.png file in the icons directory. eg for the example above, you would provide a property-shadow.png file in the icons directory.

You need to define the size and anchor of the icons used in the javascript config. All marker icons must be the same size.

8.4 Batch Geocoding

POI Auto List provides an aditional tool to retrive the latitude and longitude of all database records that do not currently have one set. Without a latitide and longitude, the point will not be shown.

Once you have completed you database configuration, you can open the db/geo.php file. This will search for all records without a latitude set and update it accordingly. There may be some cases where the batch geocoder cannot resolve the address. In this case, you will be shown which records fail and you should check the addresses and either simplify them, or enter manually.

The Batch geocoder uses the Google geocoding service. This service allows you to geocode somewhere between 2500 and 15000 addresses per day. If you need to do more than that, you will need to continue over a number of days or use some other method.
